How It Works (Simple Dimension Encoding):

  • 12-bit Encoding: Tracking ID split into two 6-bit values
  • Vertical Crop (Height): High 6 bits (bits 11-6) = 0-63 pixels removed from height
  • Horizontal Crop (Width): Low 6 bits (bits 5-0) = 0-63 pixels removed from width
  • Example: ID 1234 = binary 010011010010
  •   → Vertical: 010011 = 19 pixels from height
  •   → Horizontal: 010010 = 18 pixels from width
  • Decoding: Just compare original vs current dimensions
  • No markers needed: The dimensions ARE the data
